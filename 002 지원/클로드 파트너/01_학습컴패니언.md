## 🎯 코스 3: Introduction to Model Context Protocol (MCP)

**URL:** https://anthropic.skilljar.com/introduction-to-model-context-protocol
**구성:** 4섹션 (예상 1시간)

### 사전 핵심 (영상 보기 전 5줄)

1. **MCP = Model Context Protocol.** AI(클로드)에게 외부 도구·데이터를 연결하는 **표준 규격**.
2. **비유: USB-C.** 예전엔 도구마다 다른 케이블 필요했음. 이제 USB-C 하나면 노트북·폰·헤드폰 다 꽂힘. MCP가 AI 세계의 USB-C.
3. **3개 역할:** **Host**(클로드 앱 = 사용자 인터페이스) ↔ **Client**(연결 담당 직원) ↔ **Server**(도구 보유자: Notion, GitHub, Slack 등).
4. **Server가 제공하는 3가지:** **Tools**(클로드가 호출하는 함수, 예: "노션 페이지 만들기"), **Resources**(읽기용 데이터, 예: 파일 내용), **Prompts**(미리 만든 명령 템플릿).
5. **Transport(전송 방식):** **stdio**(내 컴퓨터에서 직접 실행), **HTTP/SSE**(원격 서버). 둘 다 메시지 형식은 똑같음 (JSON-RPC).

### 디자인씽킹 비유 (키키님 전공 연결)

MCP는 **워크숍 진행자 + 참가자 + 도구 보드**의 관계와 같음.
- **Host (클로드 앱)** = 워크숍 진행자가 서 있는 무대
- **Client** = 진행자가 들고 있는 무전기 — 참가자(서버)들과 동일한 주파수로 통신
- **Server (Notion MCP, GitHub MCP 등)** = 각각 다른 도구를 들고 온 참가자들 (한 명은 포스트잇, 한 명은 화이트보드, 한 명은 카메라)
- **MCP 표준** = 모두가 같은 무전기 주파수와 같은 손 신호를 쓰자고 약속한 규칙
- 약속만 지키면 → 새 참가자(새 서버) 누구든 무대에 와서 즉시 협업 가능

### 자주 나오는 용어 (모르면 즉시 물어보기)

- **JSON-RPC** = "JSON으로 함수 호출하기" 약속. 메뉴판에서 음식 주문하듯 `{"method": "주문", "params": {"메뉴": "김밥"}}`.
- **stdio** = standard input/output. 같은 컴퓨터 안에서 두 프로그램이 입·출력 통로로 대화하는 방식.
- **SSE / HTTP streaming** = 서버가 답변을 한 번에 안 주고 조금씩 흘려주는 방식 (실시간 자막처럼).
- **Tool calling / Function calling** = 클로드가 "이 도구 좀 써줘"라고 시스템에 요청하는 동작.
- **Schema** = "이 도구는 이런 모양의 입력을 받고 이런 모양의 출력을 줍니다" 사용설명서.

### 실습 시뮬레이션 가이드

코스에서 "MCP 서버를 직접 만들어보세요" 같은 실습이 나오면:
- 키키님: 실습 지문 + 빈칸 코드 붙여넣기
- 나: 정답 코드 + 실행 결과를 이 세션에서 시뮬레이션해서 출력
- 따로 API 키, 결제 불필요

## 🎯 코스 4: Claude Code in Action

**URL:** https://anthropic.skilljar.com/claude-code-in-action
**구성:** 4섹션 (예상 1시간 — **키키님이 지금 쓰고 있는 도구라 가장 빠름**)

### 사전 핵심 (영상 보기 전 5줄)

1. **Claude Code = 터미널/IDE에서 일하는 코딩 파트너 클로드.** 지금 키키님이 쓰는 이 도구.
2. **슬래시 명령(/help, /clear 등)** = 단축키 메뉴. **CLAUDE.md** = 프로젝트별 클로드한테 미리 알려줄 규칙 적어두는 메모장.
3. **서브에이전트(subagent)** = 특정 일에 특화된 전문가 클로드. 본체 클로드가 "이건 explore 에게 시키자" 하고 외주.
4. **MCP 서버 연결** = 클로드 코드에 외부 도구(Notion, GitHub) 꽂기. 코스 3 내용이 여기서 실전.
5. **Hooks** = 어떤 이벤트(파일 저장 등) 일어나면 자동으로 명령 실행하는 자동화. (코스에선 가볍게 소개)

### 키키님 강점

- **이미 일상에서 사용 중** → 영상의 모든 장면이 익숙할 것. 1.5x로 봐도 따라감.
- 실습이 "터미널에서 claude 명령 입력" 류면 → 진짜 해보거나, 그냥 이 세션에서 "이렇게 동작합니다" 시뮬레이션 OK.

## 🎯 코스 2: Building with the Claude API (보스, 진행 중)

**URL:** https://anthropic.skilljar.com/claude-with-the-anthropic-api
**구성:** 9모듈 (예상 4.5시간)
**현재 상태:** 진행 중 — 어디까지 했는지 확인 필요.

### 사전 핵심 (영상 보기 전 큰 그림)

이 코스는 **"개발자가 코드로 클로드를 부리는 방법"**을 가르침. 비개발자 키키님이 어렵게 느낄 수 있는데 — **개념만 잡으면 퀴즈 통과 가능**, 실제 코드 작성은 내가 시뮬레이션.

| 모듈 (예상) | 핵심 개념 | 한 줄 비유 |
|---|---|---|
| 1. API 시작/Auth | API key | 클로드 호출용 출입증 |
| 2. Messages API | 대화 보내고 받기 | 채팅 한 번에 한 통씩 |
| 3. System prompt | 성격 설정 | 알바생한테 매뉴얼 주기 |
| 4. Tool use | 클로드가 도구 호출 | "계산기 좀 줘봐" |
| 5. Streaming | 답변 조금씩 흘려받기 | 실시간 자막 |
| 6. Vision | 이미지 입력 | 클로드 눈 달기 |
| 7. Prompt caching | 같은 컨텍스트 재사용 할인 | 단골 손님 멤버십 |
| 8. Extended thinking | 깊이 생각하는 모드 | "잠깐 더 고민해봐" |
| 9. Batch / Files | 한꺼번에 처리, 파일 첨부 | 택배 한 번에 묶음 |

> 정확한 모듈 제목은 키키님이 본 화면 기준으로 다시 매핑. 영상 켤 때 "현재 모듈 X 핵심" 물어보면 그 모듈에 맞춰 더 깊게.

### 실습은 전부 시뮬레이션 가능

- "이 Python 코드 실행해서 결과 보세요" → 키키님이 코드 붙여넣기 → 내가 결과 출력
- "이 system prompt를 바꿔보세요" → 내가 두 버전 비교 보여주기
- "tool use 만들어보세요" → 내가 schema부터 호출 흐름까지 보여주기
