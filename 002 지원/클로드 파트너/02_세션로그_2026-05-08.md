## 📚 uv 쉬운 설명 (복습용)

### 한 줄 핵심
**uv는 Python용 앱스토어 + 자동 실행기.**

### 일상 비유: 새 카페 알바생

**옛날 방식 (`pip`):**
1. 새 직원 뽑으려면 → 전용 사무실 빌리기 (가상환경)
2. 사무실에 책상·문구류 다 따로 사기 (패키지 설치)
3. 매일 출근 카드 찍고 들어가기 (활성화)
4. 그제야 일 시작

**uv 방식:**
- "`uvx` 일해줘" 한 마디 → uv가 사무실·장비·출근까지 자동 처리

→ **귀찮은 4단계를 1단계로 압축한 도구**

### MCP에서 왜 중요한가

코스 3 영상에서 이런 명령 자주 등장:
```bash
uvx mcp-server-fetch     # 웹페이지 가져오는 MCP 서버
uvx mcp-server-git       # Git 작업하는 MCP 서버
```

이 한 줄이 의미하는 것:
> "MCP 서버 자동 다운로드 → 격리된 공간에서 실행 → 클로드와 연결" — 모두 자동

uv 안 깔려 있으면 클로드 데스크탑 앱 설정의 `"command": "uvx"` 줄이 전부 에러남.

## 🎯 코스 3 (MCP) 사전 핵심 — 영상 보기 전 5줄

1. **MCP = Model Context Protocol.** AI에게 외부 도구·데이터 연결하는 표준 규격.
2. **비유: USB-C.** 도구마다 다른 케이블 → 이제 하나로 통일.
3. **3개 역할:** Host (클로드 앱) ↔ Client (연결 담당) ↔ Server (Notion/GitHub 등 도구).
4. **Server가 주는 3가지:** Tools (함수), Resources (읽기 데이터), Prompts (템플릿).
5. **전송 방식 2개:** stdio (내 컴퓨터 직접) / HTTP·SSE (원격). 둘 다 메시지는 JSON-RPC.

### 디자인씽킹 비유 (키키님 전공 연결)
- **Host** = 워크숍 진행자가 선 무대
- **Client** = 진행자의 무전기 (참가자들과 같은 주파수)
- **Server** = 각자 다른 도구 들고 온 참가자들 (포스트잇·화이트보드·카메라)
- **MCP 표준** = "모두 같은 주파수·같은 손 신호 쓰자" 약속
- → 새 참가자 와도 즉시 협업 가능

### 자주 막히는 용어 (다시 복습할 때 참조)
- **JSON-RPC** = JSON으로 함수 호출하기 약속. 메뉴판 주문서 모양.
- **stdio** = standard input/output. 같은 컴퓨터 내 두 프로그램 간 입출력 통로.
- **SSE** = 서버가 답변을 한 번에 안 주고 자막처럼 흘려주는 방식.
- **Tool calling** = 클로드가 "이 도구 좀 써줘" 시스템에 요청하는 동작.
- **Schema** = "이 도구는 이런 입력 받고 이런 출력 줘요" 사용설명서.
